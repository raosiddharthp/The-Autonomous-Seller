One shared platform beneath.

The AS Suite is not eight independent applications. Every module shares the same data fabric, the same HITL framework, the same XAI layer, and the same audit trail. The Platform domain is the invisible foundation that makes every other module possible.

Module Dependencies

Can you deploy one module without the others?

Yes — with one universal rule and a few module-specific caveats. The universal rule: every module requires the H1 shared platform enablers to be live before it can deploy. Beyond that, each module's specific prerequisites are listed below.

Universal prerequisite — applies to all 8 modules
H1 Shared Enablers must be live before any module deploys: HITL Framework · XAI / SHAP Layer · Pub/Sub Event Fabric · Firestore State Store · BigQuery Audit Trail · VPC-SC Perimeter · CMEK · IAM / Workload Identity. This is why Data Governance and the platform infrastructure are Horizon 1 — they are not optional prerequisites, they are prerequisites.
M-01 · Commercial · H3
CCAI Sales Agent
Can deploy standalone
Needs H1 shared enablers only. No ML Feature Store dependency — the agent is conversational, not model-driven. Conversational state in Firestore; no pre-trained model inference required.
Soft link to M-02: ContractGuard handles contract review post-qualification. Without it, contracts route to Legal manually — the Sales Agent still works fully.
M-02 · Commercial · H2
ContractGuard
Can deploy standalone
H1 shared enablers + partial Data Governance (contract metadata features in Feature Store). Can run with reduced accuracy before the Feature Store is fully populated with historical clause data.
No cross-module dependency. Fully self-contained. Does not depend on RevRec AI or any other domain module at runtime.
M-03 · Financial · H2
RevRec AI
Can deploy standalone
H1 shared enablers + Data Governance (contract features are primary model inputs — EU AI Act Art. 11 requires feature lineage). SAP integration required for the GL write. HITL framework must be proven in production before Finance Controller workflows depend on it.
Soft link to M-02: ContractGuard enriches the feature set. Without it, RevRec AI reads contract data directly from Salesforce — slightly lower feature richness but fully operational.
M-04 · Financial · H2
FinRisk Sentinel
Can deploy standalone
H1 shared enablers + Data Governance (financial features from Feature Store gate the anomaly model). BigQuery streaming inserts must be live and schema-validated before the model can score events reliably.
No cross-module dependency. Operates on financial events independently of Commercial modules. Shares the platform layer only.
M-05 · Operations · H2
Asset IQ
Hard dependency on Data Governance
H1 shared enablers + Data Governance + unified telemetry pipeline. Asset IQ is blocked until all 6 regional schemas are validated and the unified Pub/Sub pipeline is producing quality-scored feature values. This is a hard dependency — not a soft one. Model training cannot begin until the Feature Store has been running long enough to accumulate stable baseline statistics.
No cross-module dependency. Operationally isolated from Commercial and Financial modules at runtime.
M-06 · Operations · H3
GreenOps Platform
Can deploy standalone
H1 shared enablers + GKE Autopilot (for deferred job scheduling). Integrates with GCP Carbon Footprint API — no dependency on Feature Store or any domain module data. Carbon scheduling operates on job metadata, not ML inference outputs.
Soft link to all batch ML modules: GreenOps defers training jobs for RevRec AI, ContractGuard, Asset IQ. Without those modules deployed, it has no workloads to schedule. Self-contained when other modules are present.
M-07 · Platform · H3
Strategy Dashboard
Requires H2 modules to be meaningful
H1 shared enablers + BigQuery read access across all module datasets. The dashboard can deploy technically at any point, but panels are empty until the domain modules they aggregate are live. The EU AI Act compliance panel requires RevRec AI + HITL audit trail (H2). The fleet health panel requires Asset IQ (H2).
Read-only. No write operations. No HITL checkpoints. Does not affect the runtime behaviour of any module it reads from.
M-08 · Platform · H1
Data Governance
H1 prerequisite — must deploy first
This is the first module deployed in H1 — it is a prerequisite for every other module, not a peer. Feature Store schema validation, lineage tagging, and quarantine logic must be running before any domain module data flows into training or inference. The data quality score threshold (FRD-07) must be met before any ML model sees production data.
All six domain modules depend on Data Governance. It is not optional. It is the H1 platform foundation.
